Editing files

If you don't know anything about HTML and web pages, before doing anything with this template I recommend you to take at least some basic tutorial or hire somebody experienced to do this.

As for tutorials I recommend w3schools.com

Editing php and JavaScript files is really advanced stuff, only experienced person should edit them. If you're not sure, do not change anything, it might cause serious damage to the scripts. Hiring somebody, who knows what to do, is better solution.

If you are not familiar with these technologies, do not play around with these files.

Show content

Inspect element - when editing html and css files utilities like firebug (free firefox plugin) are great helpers. With them you can easily find element style, even the stylesheet name and line number where the inspected style rule is.

Edit html/css/js/php files - use your favorite text editor. Editor, that highlights syntax is good choice, Pspad(free) or Notepad++(free) might be enough.

View pages - any common browser like Mozilla Firefox(free)...

Upload on server - look for some FTP program - filezilla(free)...

For installing this template, there are a few requirements you need on your web server in order to get full functionality from Emily. You will need to contact your webhost if the following requirements are missing.

  • PHP 5.2+
  • PHP Mail Support

You must have JavaScript enabled in your browser. Also, a web server is required for the video background - testing it locally (unless using a development server such as Wamp or Mamp server) will not work.

If a user does not have JavaScript enabled in their browser, the site does not load and a message is displayed informing them of the problem.

This is not a Wordpress theme, so cannot be installed with Wordpress.

Installation

To install Emily on your web server, download the files from ThemeForest and unzip them to the local disk on your computer. Inside the extracted folder you will see two folders named Emily - Documentation and Emily - Template. Using an FTP Client (such as Filezilla or Total Commander), upload all of the contents of the Emily - Template folder to your server.

Contact form

These scripts need php support - so if you will try this on your personal computer it will probably not work (unless you have Apache server or something like that running). You have to try it on some web server and this server must have php support. PHP (and mail function) si reguired for sending emails. Most of the web servers have this support, it's basic equipment, but some might not.

Make changes based on this documentation, do not change anything else if you are not sure, what to do, you can cause serious damage to the scripts.

Do not change contact forms, unless you know for sure what to do. Elements in contact forms must correspond to the ajax variables.

After changes are made, remember to check if it works properly.

Where to start?

Contact: open file send_email.php read the comments on the top lines and make the changes. That's all for contact form.

PHP files

class.emailSender.php (include/class.emailSender.php)

PHP class for sending emails.

Sending emails: this is just class file, include it to other scripts and use it there

Editing user messages: at the top of that file you can see $user_messages and $control_messages variables. They are set for two languages - EN and CZ - here you can change the messages (only the right side, do not delete whole line, it will cause errors and do not leave the message empty - for ajax empty message means everything's ok). For changing the language find variable "$language" (orignally "$language = 'EN';") and change it to another language code that is defined.

Adding new languages: copy one of the languages made, change the language code ('EN' to 'FR' for example) and change the messages (only the right side, do not delete whole line, it will cause errors and do not leave the message empty - for ajax empty message means everything's ok). Then change the language to the new one - find variable "$language" (orignally "$language = 'EN';") and change it to another language code that is defined.

Ajax

Visitors send messages via ajax - javascript utility, that can send data to php scripts without reloading the page.

Activation is in js/custom.js

Data to send

Ajax send data from contact forms - elements in contact forms must correspond to the ajax variables.

See functions form_to_ajax_request()

Custom messages

To edit messages from php scripts see those scripts.

Editing ajax messages - these messages are set as "data" attribute of corresponding form, for example attribut "data-success-msg" contains success message from ajax.
